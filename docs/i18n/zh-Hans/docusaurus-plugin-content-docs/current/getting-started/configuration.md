# 配置

Planova 的设置页面允许你配置语言、流水线行为和 AI 提供商连接。从顶部导航栏打开设置。

## 语言

Planova 支持两种界面语言：

| 代码 | 语言 |
|------|------|
| `en-US` | English |
| `zh-CN` | 简体中文 |

点击语言按钮即可切换。更改会立即在整个 UI 中生效。

## 流水线模式

Planova 提供两种流水线模式用于解析户型图：

### 混合 CV+VLM（默认）

使用经典计算机视觉（边缘检测、轮廓分析）处理墙体和房间几何结构，结合视觉语言模型进行语义理解（房间标注、家具、门窗类型）。此模式通常能产生更准确的墙体几何形状。

### 传统模式

由 VLM 全权处理——包括几何提取和语义分析。此模式更简单，但在复杂户型图上可能产生不够精确的墙体位置。

:::tip 何时使用传统模式
如果混合模式对你的特定户型图风格产生了异常几何形状，可以尝试传统模式。一些手绘或非标准户型图在纯 VLM 方式下效果更好。
:::

## LLM 提供商配置

Planova 使用三个独立的 AI 提供商，各自服务于流水线中的不同用途。你可以独立配置每个提供商。

三个提供商共享相同的配置字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| **Base URL** | API 端点地址 | `https://api.openai.com/v1` |
| **API Key** | 你的身份验证密钥 | `sk-...` |
| **Model** | 要使用的模型标识符 | `gpt-4o` |

每个提供商卡片都有一个**测试连接**按钮，会发送一个轻量级请求来验证你的凭据并测量延迟。

### VLM 提供商（图像解析）

该提供商运行视觉语言模型，读取户型图图像并提取结构化数据：房间、墙体、门、窗户及其位置。

**要求：** 模型必须支持图像输入（多模态）。示例：
- `gpt-4o` (OpenAI)
- `claude-3.5-sonnet` (Anthropic)
- `qwen-vl-max` (Alibaba)

### Chat 提供商（家具规划）

该提供商运行语言模型，根据房间类型、大小和你选择的风格预设，决定在每个房间中放置哪些家具。

**要求：** 任何具有能力的对话/补全模型。示例：
- `gpt-4o` (OpenAI)
- `claude-3.5-sonnet` (Anthropic)
- `deepseek-chat` (DeepSeek)

### 图像提供商（AI 渲染）

该提供商根据给定的相机视角和提示词，生成 3D 场景的照片级真实感渲染图。

**要求：** 图像生成 API。示例：
- `dall-e-3` (OpenAI)
- 自托管的 Stable Diffusion 端点

:::warning 可以使用不同提供商
三个提供商不必来自同一供应商。你可以使用 OpenAI 作为 VLM、DeepSeek 用于对话、本地 Stable Diffusion 实例用于渲染——选择在每项任务中能提供最佳效果和成本的方案即可。
:::

### 测试连接

对每个提供商，填写 Base URL、API Key 和 Model 字段后，点击**测试连接**。测试将会：

- 验证端点是否可达
- 验证你的 API Key 是否被接受
- 确认模型是否存在
- 以毫秒为单位报告往返延迟

绿色对勾及延迟（如"连接成功 (342ms)"）表示一切正常。红色指示器和错误信息会帮助你定位问题。

## 数据存储位置

Planova 将所有数据存储在平台特定的应用数据目录中：

| 平台 | 路径 |
|------|------|
| Linux | `~/.local/share/com.planova.app/` |
| macOS | `~/Library/Application Support/com.planova.app/` |
| Windows | `C:\Users\<you>\AppData\Roaming\com.planova.app\` |

该目录包含：

- `planova.db` -- SQLite 数据库，存储项目、文件和设置
- `uploads/` -- 上传的原始户型图图像
- `previews/` -- 生成的预览图像
- `pipeline/` -- 中间流水线产物（按项目分类）

:::tip 备份数据
要备份所有项目，只需复制整个 `com.planova.app` 目录。SQLite 数据库是自包含的，便于迁移。
:::

## 保存设置

修改完成后，点击设置页面顶部的**保存**按钮。设置会持久化到 SQLite 数据库中，并立即生效。

:::info 设置是按安装实例存储的
所有设置（包括 API 密钥）都存储在你的本地机器上。除了你明确配置的 AI 提供商外，不会发送到任何外部服务。
:::
